## 2. How git branch names are generated (the core answer for your change)

There are two layers.

**(a) The run ID** — `main.go:371`:

```go
cfg.runID = "fm" + strconv.FormatInt(time.Now().Unix(), 36)
```

So `runID` is the literal string `fm` followed by a base-36 Unix timestamp, e.g. `fmlk3n8a`. This `fm` is currently the only "prefix" on branch names, and it's hardcoded.

**(b) The destination ref** — generated once per agent in `runLevel`, `main.go:256`:

```go
ref := fmt.Sprintf("refs/heads/%s-c%d-a%d", cfg.runID, c, i)
```

where `c` is the concurrency level and `i` is the agent index (0-based). It's passed into `newAgent(i, repoPath, node, ref, ...)` at `main.go:257` and stored on the agent as `a.ref` (`agent.go:38`, commented `// destination ref, e.g. refs/heads/&lt;run&gt;-a7`).

So the **current branch-name format** is:

```
refs/heads/fm&lt;base36-unix-ts&gt;-c&lt;concurrency&gt;-a&lt;agentIndex&gt;
```

e.g. `refs/heads/fmlk3n8a-c32-a5`.

**How the ref is used per strategy:**

- `branch` and `repo` strategies — `agent.push`, `agent.go:190-198`:
  ```go
  RefSpecs: []config.RefSpec{config.RefSpec("+refs/heads/master:" + a.ref)},
  Force: true,
  ```
  Force-pushes the local `master` to the agent's ref.

- `session` strategy — `runSessions`, `session.go:52`:
  ```go
  sessRef := fmt.Sprintf("%s-s%d", a.ref, session)
  ```
  Appends `-s&lt;sessionNumber&gt;` to the base ref, so session refs look like `refs/heads/fmlk3n8a-c32-a5-s3`. That `sessRef` is pushed (`pushRef`, `session.go:130-148`) and later deleted (`deleteRef`, `session.go:156-175`).

There is **no other branch-naming logic** and no existing user-facing prefix control — the only prefix is the hardcoded `fm` in `runID`. For a `-branch-prefix` flag, the single generation site to change is `main.go:256` (and you'd feed the new prefix through `runConfig` + `parseFlags`); everything downstream (`agent.push`, `runSessions`/`pushRef`/`deleteRef`) just consumes `a.ref` and needs no change.
